The Dataset dropdown only lists datasets in Submitted or In QA status — anything already Approved or Returned for Revision drops off automatically.

To see the full loop:

  1. Select a dataset from the dropdown
  2. Set the sampling criteria and select Generate Samples
  3. Review each one manually or use the Demo Shortcut in the bottom-right to complete the QA review
  4. Approve or return the dataset for revision
  5. Watch the dataset disappear from this dropdown once you complete the flow
  6. Repeat the process with another dataset and explore a different QA outcome workflow

Every QA review flows through two editable inputs — Ground Truth Label and Attributes — and one auto-derived output — Reviewer Assessment.

How it works:

  1. Leave both untouched → assessment locks to Accepted
  2. Change the label only → Label Corrected
  3. Toggle an attribute chip only → Attributes Corrected
  4. Change both → Label & Attributes Corrected

The highlighted row under Reviewer Assessment updates in real time as you edit, so you never pick it manually — your edits drive the selection of the Review Assement outcome. Then just add your note back to the assigned annotator and submit. The modal will automatically proceed to the next sample in the queue.
